01 — What changed in v3

Stop designing pages. Compose one system.

v1 specced the landing + generator. v2 polished them and patched tablets. They drifted — three type systems, two shells, inconsistent density. v3 collapses everything onto the Workspace dashboard so a seller feels the same product on every screen.

The five decisions that define v3

01
Workspace is canon
Every surface inherits the Workspace shell, tokens and components. New screens are assembled from existing parts, not designed fresh.
02
One type family
Newsreader is retired. Inter (+ Noto Sans Thai) carries every headline; JetBrains Mono for labels & metadata. Marketing and app now match.
03
Compact by default
The density-compact scale ships as the default for all authenticated surfaces. Cozy stays available as a user preference only.
04
One accent rule
Orange = primary action (Generate / Download / New). Indigo = selection & AI-adjacent only. Ink = everything else. No exceptions.
05
Deterministic voice
BrandBatch renders templates, it doesn't "AI-generate". The v2 vocabulary & sparkle guardrails carry forward unchanged.

03 — Density · Compact (locked default)

Compact is the shipping default.

The Workspace exposes two density modes via the same .dc-* utility hooks. v3 ships Compact as the default class on <body> / app root. Cozy remains wired as a per-user preference (Settings → Appearance) but is never the default.

🔒 Default density rule

Authenticated app root mounts with class="density-compact". Persist the user's choice in localStorage["bb_density"]; if unset, Compact. Marketing + auth pages are always Compact (no toggle).

04 — The app shell

Sidebar + TopBar — the spine of every authed screen.

Every route behind login renders inside one shell: a collapsible left Sidebar and a sticky TopBar. Only the main region changes per route. This is what makes fourteen screens feel like one product.

09 — Build order & handoff

Ship the shell, then fill it.

Don't
  • Re-design any page from scratch.
  • Add Newsreader or any third font.
  • Introduce colors outside the 17 tokens.
  • Make Cozy the default, or remove the toggle.
  • Re-architect the generator's export math.
  • Add "AI generated / Auto-frame / Smart layout" copy or extra ✦.
Build order
  • 1. Extract Sidebar + TopBar into a shared AppShell (§04).
  • 2. Promote Workspace blocks to a real component lib (§05).
  • 3. Set density-compact default + Settings toggle (§03).
  • 4. Move /dashboard onto the shell (reference build).
  • 5. Build Products, Templates, Brand Kit, Photos, Generations from the lib.
  • 6. Wrap /generator in the shell; restyle to tokens.
  • 7. Re-skin marketing (drop serif) + auth cards.
  • 8. Re-run npm run lint:vocab + build.
🔒 The one-line definition of done

A seller can move from landing → login → dashboard → any library page → generator and never feel a seam: same paper, same Inter, same Compact rhythm, same orange action, same cards.
